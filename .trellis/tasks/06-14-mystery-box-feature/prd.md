# 盲盒推荐功能

## Goal

在现有"WhatToEat-AI"微信小程序中增加盲盒推荐功能，根据用户地理位置和POI数据，通过盲盒算法自动推荐餐饮，提供更有趣和惊喜的用餐选择体验。

## Current Context

**Confirmed Facts:**
- 项目是微信小程序原生开发，用于餐饮推荐
- 现有功能：位置获取、POI数据（高德API）、AI推荐（GLM）、场景选择
- 数据源：`getPoi`云函数获取附近商家，`recommend`云函数提供AI推荐
- 前端评分算法：距离衰减、质量评分、场景匹配
- 界面：场景选择、推荐卡片、换一批功能
- 用户提供了首页截图作为参考

## Requirements

**UI/UX Requirements:**
- 首页采用左右滑动切换布局
- 左侧页面：现有AI推荐功能（保持不变）
- 右侧页面：新增盲盒推荐功能
- 默认加载左侧AI推荐页面
- 顶部或底部添加tab指示器，显示当前所在页面

**盲盒页面交互设计（基于最佳实践研究）：**
- 单抽模式：一次推荐一家餐厅，强调仪式感
- 毛玻璃盲盒：初始状态显示毛玻璃效果，隐隐透露餐厅类型
- 开盒动画：2-3秒开盒动画（晃动→光芒→毛玻璃消散）
- 震动反馈：配合wx.vibrateShort增强体验
- 历史记录：底部展示本次会话开过的盲盒（横向滚动）
- 操作选项：导航、收藏、再开一次

**盲盒次数机制（Confirmed）：**
- ❌ 不设每日次数限制（本地算法无API成本）
- ✅ 会话内去重：本次使用期间不重复推荐同一家
- ✅ 2秒冷却：配合动画节奏，防误触
- ✅ POI池耗尽提示：附近店开完后提示并引导换位置

**盲盒推荐算法（Confirmed - 基于E&E最佳实践）：**
- 算法选型：Epsilon-Greedy + 长尾加权 + 会话去重
- 理论依据：E&E问题（探索与利用），盲盒本质是"探索"的极端体现
- 算法流程：
  1. 质量门槛：评分≥3.5 或（无评分且距离≤500m），且距离≤3km
  2. 会话去重：排除本次已开过的poi_id
  3. Epsilon-Greedy决策（ε=0.3）：
     - 30%概率：纯随机探索（从候选池均匀随机抽1家）
     - 70%概率：加权利用（按weight加权随机抽1家）
  4. 权重公式：weight = 0.4×distanceScore + 0.4×qualityScore + 0.2×longTailBonus
- 纯前端算法，无API成本
- 与AI推荐差异化：AI是贪心选Top(100%利用)，盲盒是30%探索+70%利用

**数据来源与复用策略（Confirmed）：**
- 完全复用现有 `getPoi` 云函数返回的 pois 数据
- 盲盒页面与AI推荐页面共享 `this.data.pois`
- 不新增云函数，盲盒算法纯前端运行
- 用户切换页面无需重新请求网络
- "换位置"时两个页面数据同步更新

**推荐理由方案（Confirmed）：**
- MVP采用模板理由，不调用AI（GLM API）
- 盲盒专属文案，带emoji和"盲盒"关键词
- 突出"惊喜""幸运""随机"的盲盒调性
- 区别于AI推荐的严肃理由
- 示例：`🎁 恭喜开出XX！4.5分的好店`

**用餐场景处理（Confirmed）：**
- 盲盒不考虑用餐场景选择，纯随机探索
- 与AI推荐差异化：AI理性匹配场景，盲盒感性随机
- 强化盲盒"未知惊喜"的本质

**风险处理：时段不匹配（Confirmed）：**
- 采用三层处理策略：软引导 + 硬提示，不做硬过滤
- 第1层：时段感知加权（匹配时段×1.3，不匹配×0.7）
- 第2层：复用SCENE_KEYWORDS识别餐厅场景
- 第3层：不匹配时友好提示（如"🌙 这是家早餐店，晚上可能不营业"）
- 保留盲盒惊喜感，不硬过滤任何店

**Confirmed Design Decisions:**
- 用户明确要求左右切换布局，而非独立按钮或页面
- 盲盒推荐作为独立页面与AI推荐并列
- 盲盒推荐采用纯本地算法，无次数限制
- 算法基于E&E理论最佳实践（Epsilon-Greedy）
- 数据复用现有getPoi，无新增云函数
- 推荐理由用模板生成，不调用AI
- 盲盒不考虑用餐场景，纯随机探索
- 时段不匹配用软引导+硬提示处理

## Acceptance Criteria

### UI/交互验收
- [ ] 首页支持左右滑动切换，左侧AI推荐、右侧盲盒推荐
- [ ] 默认加载左侧AI推荐页面
- [ ] 有明确的tab指示器显示当前页面
- [ ] 盲盒页面显示毛玻璃盲盒图标和"开盲盒"按钮
- [ ] 点击开盲盒播放2-3秒开盒动画
- [ ] 开盒时有震动反馈（wx.vibrateShort）
- [ ] 开盒后展示一家餐厅完整信息卡片
- [ ] 底部展示本次会话历史记录（横向滚动）
- [ ] 操作按钮：导航、再开一次可用

### 算法验收
- [ ] 盲盒推荐从 `this.data.pois` 中选取，不调用云函数
- [ ] 质量门槛生效：评分≥3.5 或（无评分且距离≤500m），距离≤3km
- [ ] 会话内去重：同一会话不重复开出同一家店
- [ ] Epsilon-Greedy：约30%概率随机，70%概率加权
- [ ] 2秒冷却：连续点击有防抖
- [ ] POI池耗尽：所有店开完后显示提示并引导换位置

### 场景处理验收
- [ ] 时段感知加权：匹配当前时段的店更易被开出
- [ ] 时段不匹配时显示友好提示文案
- [ ] 不做硬过滤：任何质量达标的店都可能被开出

### 数据一致性验收
- [ ] 盲盒页面与AI推荐页面共享同一份pois数据
- [ ] "换位置"后两个页面数据同步更新
- [ ] 切换页面不触发网络请求

### 推荐理由验收
- [ ] 盲盒理由使用模板生成，不调用GLM API
- [ ] 理由文案带emoji和盲盒调性（🎁✨🎲等）
- [ ] 与AI推荐理由有明显区别
