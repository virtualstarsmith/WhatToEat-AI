# AI甄选快捷筛选栏

## 背景

PRD §3.3：AI甄选被吐槽"推荐不够好"。PRD 原方案的 C4（个性化权重）因无用户画像/实时热度数据源被砍，C2（排队状态）因无数据源被砍。本任务用"手动挡的个性化"——快捷筛选栏——解决"推荐不够好"的真痛点：与其让 AI 猜用户想要什么，不如让用户直接筛。

## 需求

### 1. 筛选栏位置与形态

- 位于首页（index）列表上方、场景栏下方
- 快捷标签形式（chip / tag），点击切换，支持单选或互斥的多档位

### 2. 筛选维度（三个，不含口味）

| 维度 | 数据源 | 档位示例 |
|---|---|---|
| 价格（人均） | `poi.cost` | ¥30内 / ¥50内 / 不限 |
| 距离 | `poi.distance` | 500m内 / 1km内 / 不限 |
| 类别 | `poi.type` | 快餐 / 正餐 / 不限 |

- **不含口味（辣/不辣）**：菜系推断误差大（川湘=辣的假设不可靠），评审已决策不做。

### 3. 筛选逻辑

- 在 `scoreCandidates` / `topNWithExplore` 前加一层 filter，按当前选中档位过滤候选池
- 筛选状态变化时重置推荐结果（类比现有切场景逻辑：清空 recommendations/cardsView + 重新 callRecommend）
- 筛选与现有 exclude（换一批去重）机制共存，不冲突

### 4. 与 AI 推荐的协作

- 筛选后仍走 AI 推荐流程（callAIRecommend），AI 从过滤后的候选池中挑
- 兜底路径（_useFallbackRecommend）同样基于过滤后的池子

## 涉及文件

- `pages/index/index.wxml`（筛选栏 UI）
- `pages/index/index.wxss`（筛选栏样式）
- `pages/index/index.js`（筛选状态 + 过滤逻辑 + 切换重置）
- `utils/scoring.js` 或新增 filter 工具（filter 层可放此处复用）

## 验收标准

- [ ] 首页场景栏下方显示筛选栏，含价格/距离/类别三组
- [ ] 点选"¥30内"，列表只出现人均 ≤30 的店（无 cost 字段的店的处理需明确，见 design）
- [ ] 切换筛选后列表刷新，旧推荐清空，无错乱
- [ ] 筛选与"换一批"共存：换一批在当前筛选档位内去重
- [ ] 筛选栏不破坏现有场景栏、loading、兜底等逻辑

## Notes

- 复用 A 任务（Tab 重构）不冲突：本任务只改 index 页内容区，不动 tabBar。
- 这是复杂任务，需补 `design.md`（filter 层位置、cost 为空的店如何处理、筛选与 exclude 的交互、单选/多档位互斥规则）和 `implement.md`。
- 类别（快餐/正餐）的映射规则需在 design 中明确：`poi.type` 的哪些值归"快餐"、哪些归"正餐"、其余归"不限"或不显示。
