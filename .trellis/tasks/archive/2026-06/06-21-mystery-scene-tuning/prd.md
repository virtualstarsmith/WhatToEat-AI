# 盲盒时段加权与场景关键词表优化

## Goal

修复盲盒推荐（`utils/mysteryBox.js`）的时段感知加权过强、场景关键词表覆盖不全两个遗留问题，使"高质量近距离好店"不会仅因品类词未命中而被远处的匹配店反超。本任务是 `06-14-mystery-box-feature` review 后的遗留项，源自 `design.md` 的"遗留观察"。

## Background（问题溯源）

上一轮 review（2026-06-20）修复 poi_id / 连锁降权 / 中段探索 / 无评分门槛后，验证脚本的权重诊断暴露了两个 review 范围外的既有问题：

1. **时段加权过重**：`timeAwareMultiplier` 匹配 ×1.3 / 不匹配 ×0.7，惩罚不对称且乘在总分上。
   - 实测：午餐时段「特色小面馆」(150m, 4.6分) 综合权重 `0.630`，反而低于 800m 的「阿婆肠粉」`0.919`。
   - 原因：150m 近店因"面馆"未命中午餐关键词 → ×0.7 压制；"肠粉"命中"粉" → ×1.3 加成。
2. **关键词表覆盖不全**：午餐词表 `['快餐','简餐','面食','粉','便当','盖饭']` 不含「面馆」「米线」「麻辣烫」等高频午餐品类，导致大量正常午餐店被判不匹配。

## Decisions（已与用户确认）

| 决策点 | 选择 | 说明 |
|--------|------|------|
| 时段加权系数 | 匹配 ×1.2 / 不匹配 ×0.85 | 弱化惩罚，保留软引导；未命中好店不再被严重压制 |
| 关键词表 | 扩充各场景常见品类词 | 不含连锁品牌名（连锁已由 longTailBonus 处理），不改匹配算法 |

**不改动的部分**（明确排除）：
- 匹配算法仍是子串 `indexOf`（不引入词边界匹配，控制改动面）。
- 不把连锁品牌加入 SCENE_KEYWORDS。
- `longTailBonus`、`qualifyFilter`、`midBandPick` 等上一轮已定型的逻辑不动。

## Requirements

### R1 · 调整 timeAwareMultiplier 系数

`utils/mysteryBox.js` 中 `timeAwareMultiplier`：
- 匹配：`1.3` → `1.2`
- 不匹配：`0.7` → `0.85`
- `keywords` 为空（随便吃点/未知场景）仍返回 `1.0` 不变。
- 同步更新函数注释里的系数说明。

### R2 · 扩充 SCENE_KEYWORDS 各场景品类词

`config/sceneKeywords.js`，在保留现有词条基础上补充高频品类词（覆盖此前会误判不匹配的常见店铺）：

- **早餐**：补充 `煎饺`、`饭团`、`豆花`、`馄饨`、`烧麦`、`生煎`、`锅贴`、`胡辣汤`（现有：早餐/包子/粥/豆浆/油条/肠粉/面/粉）
- **午餐**：补充 `面馆`、`米线`、`馄饨`、`水饺`、`饺子`、`酸辣粉`、`黄焖鸡`、`麻辣烫`、`砂锅`、`炒饭`（现有：快餐/简餐/面食/粉/便当/盖饭）
- **下午茶/饮品**：补充 `蛋糕`、`面包`、`甜品店`、`水果茶`、`柠檬茶`、`冰淇淋`（现有：奶茶/咖啡/甜品/烘焙/果汁/茶饮/轻食）
- **晚餐**：补充 `烤鱼`、`川菜`、`湘菜`、`粤菜`、`西餐`、`日料`、`寿司`、`海鲜`、`私房菜`、`小炒`（现有：正餐/火锅/烧烤/炒菜/饭店）
- **夜宵**：补充 `鸭脖`、`炸鸡`、`毛豆`、`螺蛳粉`、`烤鱼`、`串串`（现有：烧烤/小龙虾/粥/串/烤/宵夜）

### R3 · 回归验证

用临时 Node 脚本验证（用完即删，不引入测试框架）：
1. 上一轮的回归用例：午餐时段「特色小面馆」(150m, 4.6) 综合权重 > 800m「阿婆肠粉」（验证 R1+R2 联合效果修复了原问题）。
2. 系数断言：匹配 ×1.2、不匹配 ×0.85、空场景 ×1.0。
3. 关键词命中断言：「面馆」命中午餐、「烤鱼」命中晚餐且夜宵、「螺蛳粉」命中夜宵。
4. 关键词去重：每个场景词表无重复项。
5. `qualifyFilter` / `midBandPick` / `makePoiId` 等既有行为不受影响（跑一遍上一轮的代表性断言）。
