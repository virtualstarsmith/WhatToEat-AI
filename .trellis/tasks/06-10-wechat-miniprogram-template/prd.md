# AI今天吃什么 - MVP 产品需求文档

> 基于 `docs/AI今天吃什么-PRD初稿.md` 和当前规划决策整理。MVP 聚焦真实定位、附近商家获取、AI 推荐决策。

## Goal

构建"AI今天吃什么"微信小程序 MVP，让用户基于当前位置快速获得附近可去吃的商家推荐。

核心闭环：

1. 用户打开小程序
2. 授权真实定位
3. 小程序通过高德地图获取附近餐饮商家
4. 大模型结合距离、评分和场景推荐 1-3 家
5. 用户查看推荐理由，选择去吃、换一批，或点击导航/优惠入口

## Confirmed Decisions

- MVP 使用真实定位，不使用纯 mock 作为主路径
- 附近商家数据源使用高德地图
- MVP 的核心对象是"附近商家"，不是本地菜品菜单
- 采用微信小程序 + 云函数 + 高德 API + AI 服务架构
- 高德 API Key 和 AI 服务 Key 只放在云函数环境中，不暴露给小程序前端
- MVP 大模型服务选型为 GLM，默认使用 `glm-4.7`
- 保留 `GLM_MODEL` 环境变量配置项，可灰度切换到 `glm-5.1` 做推荐质量对照
- MVP 加入轻量商业化能力：推荐结果优惠入口
- 转盘、翻卡片、自定义菜单不进入 V0.1 主范围
- MVP 不做预算功能，推荐逻辑不考虑预算筛选
- 商业化入口 MVP 先用手动配置链接（manual），不对接第三方 CPS
- MVP 不做口味偏好/忌口设置，推荐逻辑不考虑用户偏好
- MVP 不做喜欢/不喜欢反馈
- MVP 不做独立商家详情页，导航和复制地址直接放在推荐卡片上

## Requirements

### FR-01 定位授权

- 首页进入后提示用户授权当前位置
- 使用微信小程序定位能力获取经纬度
- 用户拒绝定位时，展示明确降级提示
- 用户可手动重新触发定位授权

### FR-02 高德附近餐饮商家

- 基于用户经纬度请求高德地图附近餐饮 POI
- 默认搜索半径：2 公里，参数可调
- 商家信息至少包含：
  - 商家名称
  - 地址
  - 距离
  - 经纬度
  - 品类
  - 评分（如高德返回）
  - 人均价格（如高德返回）
- 高德接口失败时展示错误状态和重试入口

### FR-03 大模型推荐附近吃什么

- 用户选择用餐场景：早餐、午餐、下午茶/饮品、晚餐、夜宵、随便吃点
- 场景默认根据当前时间段自动填充，用户可手动切换
- 系统调用大模型，从附近商家候选集中推荐 1-3 家
- MVP 默认模型：GLM `glm-4.7`
- 灰度/对照模型：GLM `glm-5.1`（通过 `GLM_MODEL` 环境变量切换）
- 规则预筛选后取评分 top 15 家传入大模型，控制 token 成本
- 云函数向大模型传入候选商家和用餐场景
- 大模型输出必须是结构化 JSON，且推荐商家 ID 必须来自候选商家列表
- 推荐结果展示：
  - 商家名称
  - 品类/标签
  - 距离
  - 人均价格
  - 评分
  - 推荐理由
  - 导航按钮（调用 wx.openLocation）
  - 复制地址按钮（调用 wx.setClipboardData）
  - 优惠/团购入口（如有配置）
- 大模型必须输出推荐排序和自然语言推荐理由
- 规则评分只用于候选商家预筛选、成本控制和接口失败兜底
- 大模型调用失败或输出解析异常时，使用规则评分 top 3 作为兜底推荐，展示"智能推荐暂不可用，已为你按距离和评分筛选"提示

### FR-04 换一批推荐

- 用户点击"换一批"后重新排序并返回新的 1-3 家推荐
- 同一轮内尽量避免重复推荐刚展示过的商家
- 附近商家不足时允许重复，但需提示"附近可选商家较少"

### FR-05 轻量商业化入口

- 推荐卡片展示优惠/团购/下单入口（如有配置）
- MVP 使用手动配置链接（manual）作为推广来源，不对接第三方 CPS
- 推荐排序不因商业化入口而强制置顶，避免损害推荐可信度
- 推荐排序不因商业化入口而强制置顶，避免损害推荐可信度
- MVP 不做 CPS 成交回传，不做点击记录

## Recommendation Logic V0.1

MVP 推荐分两层：

1. 规则预筛选：从高德 POI 结果中选出候选商家，减少大模型输入规模
2. 大模型推荐：基于候选商家和用餐场景输出最终 1-3 家推荐及理由

规则预筛选评分（取 top 15 传入大模型）：

采用两层结构：基础分 x 场景乘子（参考美团/大众点评的乘性门控与指数衰减实践）。

    base_score = 0.50 * distance_score + 0.50 * quality_score
    final_score = base_score * scene_multiplier

评分维度说明：

- `distance_score`：指数衰减 exp(-distance/800)，800m 约步行 10 分钟。非线性符合人对步行距离的感知
- `quality_score`：高德评分归一化到 0-1，无评分时取 0.3（降权而非中位数，避免无评分商家虚高）
- `scene_multiplier`：场景与品类匹配乘 1.0，不匹配乘 0.5。乘性而非加法，防止品类不匹配的商家靠距离+评分优势挤掉匹配场景的商家。早餐(5-10h)匹配早餐/包子/粥/豆浆，午餐(10-14h)匹配快餐/简餐/面食，下午茶/饮品(14-17h)匹配奶茶/咖啡/甜品/烘焙，晚餐(17-21h)匹配正餐/火锅/烧烤，夜宵(21h-次日5h)匹配烧烤/小龙虾/粥店，随便吃点全部品类等权。"减脂"已从场景体系移除，规划作为后续偏好筛选项（与场景乘子正交）

降权规则：

- 最近一轮已推荐过的商家 final_score 乘 0.6

兜底策略（大模型不可用时）：

- 规则评分降权后取 top 3 作为推荐结果
- 展示提示文案："智能推荐暂不可用，已为你按距离和评分筛选"
- 兜底结果仍展示商家名称、距离、品类、评分，但不包含推荐理由
